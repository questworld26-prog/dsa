# Two pointer
Linear Scan Optimization

Instead of using nested loops $O(n^2)$, this pattern uses two pointers to iterate through a data structure (usually a sorted array or a linked list) from different directions or speeds to solve the problem in $O(n)$ time.

**Process**: 

Start with two pointer pointing at the opposite ends or towards the start. Change the value of the pointer based on the required condition.  

**How to recognise this pattern?**

* Data is sorted 
* Need a pair or sub array as the output
* The brute force approach involves nested loops ($O(n^2)$ or $O(n^3)$), but the constraints demand $O(n)$ time and $O(1)$ space
